Исходя из результатов (Test1.txt - Test3.txt), при увеличении размера популяции время работы алгоритма возрастает (Text1.txt).
Однако, при увеличении размера поуляции может сократиться искомая цепочка слов (Test2.txt).
При поиске обратной последовательности, найденая цепочка слов сильно возрастает (Test3.txt), что говорить о неоптимальности функции оценки похожести слов.
При уведичениии размер словаря (т.е. при у величении числа возможных цепочек), найденная цепочка слов тоже сильно возрастает (Test2.txt), что так же 
говорит о неоптимальности функции оценки похожести слов.
